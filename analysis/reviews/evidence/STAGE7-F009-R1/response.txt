## Findings (ordered by severity)

### F1 — Medium — Claimed financial-failure-injection diagnostic test is absent (FR-014 / D-006)
- **Evidence:** tasks.md marks `T001` (financial rollback **logging** tests) and `T002` (atomic financial-failure **diagnostics**) as `[x]`; plan.md:22-23 requires "Financial failure injection verifies transaction rollback and correlated diagnostics." The only test added in this slice is `OperationalApiTests.cs` (liveness/readiness, bank identity, correlation-ID). No new test injects an *unhandled infrastructure* failure (e.g., forced `DbUpdateException` mid-`ExecuteSerializableAsync`) and asserts (a) no domain/audit rows persist and (b) a correlated, sensitive-data-free diagnostic is emitted. Existing `InternalTransferApiTests.cs:76-104` exercises only validation-path rollback, not infrastructure failure.
- **Impact:** FR-014's "emit correlated diagnostics on unhandled infrastructure failure" is unverified for this slice, and the `[x]` completion of T001/T002 is an unsupported parity claim. Atomicity itself is met by pre-existing code, so this is a verification/traceability gap rather than a runtime break.
- **Correction/test:** Add an integration test that substitutes a repository/DbContext throwing on `SaveChanges` inside the serializable transaction; assert `BookedTransactions`/audit are empty afterward and that the emitted log carries the correlation ID with no body/credential/query content. Otherwise revert the T001/T002 checkmarks pending that evidence.

### F2 — Low — Broken regex makes a delivery gate a permanent false-negative (FR-002 assurance)
- **Evidence:** `verify-delivery.mjs:20` — `!api.match(/depends_on:[\\s\\S]*?setup:/)`. In a regex **literal**, `\\s`/`\\S` match a literal backslash / `s` / `S`, not whitespace/any-char. The `api` block contains no backslashes, so this assertion can never match and "startup has no setup dependency" always passes — even if `setup` were later added to `api.depends_on`.
- **Impact:** The gate gives false assurance for FR-002; a future regression would slip through the 11-control delivery audit.
- **Correction:** Use single backslashes `/depends_on:[\s\S]*?setup:/`, or (more robust) extract the `api` service's `depends_on` keys and assert `setup` is absent.

### F3 — Low — Secret scanner has known blind spots / workflow exclusion (FR-004)
- **Evidence:** `scan-secrets.mjs:15` only flags quoted values ≥16 chars keyed by `password|token|secret`; it misses PEM/private-key blocks, base64 key material, unquoted assignments, and inline `Password=` in connection strings. `.github/workflows/` is fully excluded (line 8), so the CI-embedded literals in `modern-feature-001.yml` (`Validation-*-Password-2026!`) bypass scanning entirely.
- **Impact:** False-negative risk; acceptable for a demo gate but currently undocumented.
- **Correction:** Add patterns for PEM headers and connection-string credentials, and document the workflow-exclusion limitation in OPERATIONS.md.
